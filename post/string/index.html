后缀自动机（SAM）

概述

SAM 是一种很优秀的字符串数据结构。它可以将一个字符串的所有子串压缩成一个 O(n)O(n)O(n) 大小的结构。

定义

SAM 是有限状态自动机，换种说法就是：

  • SAM 是一个有向无环图。节点被称为状态。边有边权，权值为一个字母，边称为转移。
  • 存在一个源点 t0t_0t0​，即初始状态。从 t0t_0t0​ 出发可以到达所有状态。
  • 存在若干个终止状态。从 t0t_0t0​ 到某个终止状态的路径，边权拼起来一定是 sss 的一个后缀。反之，sss 的任意一个后缀对应一条从 t0t_0t0​ 到某个终止状态的路径。

SAM 与子串的关系

从 t0t_0t0​ 出发的任意一条路径，边权拼起来一定是 sss 的子串。反之，sss 的任意子串对应从 t0t_0t0​ 出发的一条路径。

t0t_0t0​ 到每个点可能有多条路径，也就是说每个状态对应一些子串的集合。

引入其它概念

endpos

假设 sss 有非空子串 ttt，则 endpos⁡(t)\operatorname{endpos}(t)endpos(t) 表示 ttt 在 sss 中所有出现的结束位置。例如 s=abcbcs=\texttt{abcbc}s=abcbc，则 endpos⁡(bc)={3,5}\operatorname{endpos}(\texttt{bc})=\{3,5\}endpos(bc)={3,5}。

某几个子串的 endpos⁡\operatorname{endpos}endpos 可能相同，例如 s=abcbcs=\texttt{abcbc}s=abcbc，则 endpos⁡(c)=endpos⁡(bc)={3,5}\operatorname{endpos}(\texttt{c})=\operatorname{endpos}(\texttt{bc})=\{3,5\}endpos(c)=endpos(bc)={3,5}。endpos⁡\operatorname{endpos}endpos 相同的子串会被划分到同一个等价类 中， sss 的所有子串会被划分成若干个等价类。

SAM 的每个状态对应一个等价类中的所有子串。也就是说，SAM 的状态数为等价类的数量 +1+1+1，其中 +1+1+1 加上的是 t0t_0t0​，t0t_0t0​ 对应的是空串。

引理 1：u,wu,wu,w 是字符串的两个子串，则：uuu 是 www 的后缀   ⟺  \iff⟺ endpos⁡(w)⊆endpos⁡(u)\operatorname{endpos}(w) \subseteq \operatorname{endpos}(u)endpos(w)⊆endpos(u)。

引理 2：在一个等价类中，短的字符串是长的字符串的后缀，且这些字符串长度连续。

后缀链接 link

现有状态 uuu，假设 uuu 所对应的等价类中，长度最长的是 ttt。将 ttt 的所有后缀按长度递减排序，则这个等价类中包含前几个后缀。设第一个不属于这个等价类的后缀为 www（一定存在，因为 www 可以为空），则 link⁡(u)\operatorname{link}(u)link(u) 连向 www 所属的等价类所对应的状态。

所有的后缀链接构成一棵根节点为 t0t_0t0​ 的树。

注意后缀链接 link 的边与 SAM 的边并不相同。

由引理 1 可知，endpos⁡(u)⊊endpos⁡(link⁡(u))\operatorname{endpos}(u)\subsetneq\operatorname{endpos}(\operatorname{link}(u))endpos(u)⊊endpos(link(u))。

其它定义

对于状态 uuu，其等价类中最长的字符串为 longest⁡(u)\operatorname{longest}(u)longest(u)，其长度为 len⁡(u)\operatorname{len}(u)len(u)。最短的字符串为 shortest⁡(u)\operatorname{shortest}(u)shortest(u)，其长度为 minlen⁡(u)\operatorname{minlen}(u)minlen(u)。

可以得出：minlen⁡(u)=len⁡(link⁡(u))+1\operatorname{minlen}(u)=\operatorname{len}(\operatorname{link}(u))+1minlen(u)=len(link(u))+1。
